Blog 部署

概述

由 Github 托管 Blog 代码，并使用 Github Actions 向 GitHub Pages 和 Cloudflare Pages 推送，做到私有代码库，公网访问。

博客框架选择

因为使用 Gihtub Pages 和 Cloudflare Pages 所以选选择了hexo 来搭建 blog。

工作流程

初始化

准备

使用的是 windows 电脑

    1. Hexo.exe
    1. git
    1. Cloudflare 账户和域名
    1. VsCode
    1. npm
    1. yarn

开始

    1. 在 Github 上创建2个仓库：
      • remote-blog
      • xxx.github.io
    1. 将remote-blog 仓库 git clone 到本地。
    1. 打开 Powershell，将工作路径定到remote-blog，输入 将仓库初始化，后面再调用hexo.exe 将写为hexo。
    1. 输入 使用 npm 下载相关组件
    1. 创建remote-blog/.github/workflows/xxx.yml文件，用来调用 Github Actions 自动化部署
  • 6.
    1. 将修改 push 到 Github 上，此时应该能正常调用 Github Actions 但是因为缺少参数无法继续。
      [Blog_Token]:


      在这里生成一个拥有权限的Token

      [CF_ID]:

    1. 到此基础 Hexo 博客已经搭建完成了。后续还可以对 Hexo 使用主题和插件等方便使用。

部署

因为使用的是 Github Actions 自动化部署，所以在本地驶入 git push 并回车就会触发 Gihub Actions 推送到 Github Pages 和 Cloudflare Pages。
